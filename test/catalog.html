Olbi

Olbi とは

Olbi はデータをHTMLタグへうめこむことを目的にしたライブラリです。

単純な構成のページでの利用を想定した、シンプルなライブラリです。

手軽に使えることを考え、トランスパイルせず使えます。

Olbi を動作させるための基本的なステップ

Olbi を使いデータをHTMLタグにうめこむには、次の3つステップが必要です。

  • データ構造を Olbi へ宣言する
  • Olbi のデータ項目を Element に結びつける
  • データを Olbi へ設定する

3つのステップについて、単純な例を使って説明します。

データとタグ

ステップの説明をする前に、例で対象にするデータとタグを示します。 


var data = {
  party: {
    person: {
      name: "Ann"
    }
  }
};
        

  <div class="party person">
    <span class="name"></span>
  </div>

例では、データにある "Ann" を、タグの <span class="name"></span> にうめこむことをめざします。

データ構造を Olbi へ宣言する

最初のステップです。 データ構造を Olbi へ宣言します。 Olbi はデータを扱うために、データ構造を要求します。 先に示したデータと同じ構造のオブジェクトリテラルを、 コンストラクタの引数へ指定し、 Olbi にデータ構造を宣言します。

Olbi コンストラクタは、受けっとったデータ構造と同じプロパティを、インスタンスに作成します。 作成したプロパティは、次のステップで使います。 


          var olbi = new Olbi({
            party: {
              person: {
                name: ""
              }
            }
          });

Olbi のデータ項目を Element に結びつける

2番めのステップです。 Olbi のデータ項目を Element に結びつけます。 Olbi は次のコードで、 セレクター文字列生成と Element 選択、 Element とデータ項目の結びつけをおこないます。 


          olbi.party.person.name.to.text();

セレクター文字列の生成は toプロパティの呼び出しでおこないます。 to プロパティの前にある、 party プロパティと person プロパティ、 name プロパティを元に 次のセレクター文字列が生成されます。 


          ".party.person .name"

セレクター文字列はプロパティ名からクラスセレクター、 つまり、ドット "." ではじまるセレクターを生成します。 注意点があり、セレクター文字列の ".party" と ".person" のあいだにはスペースがありませんが、 ".person" と ".name" のあいだにはスペースがあるところです。 Olbi はプロパティの型が Object のときは、 親になる Object のプロパティと、 クラスセレクターを連結して、あいだのスペースはなしで生成します。 プロパティ型が String や Number、null のときは、 親になる Object のプロパティと、 クラスセレクターを子孫結合子(Descendant combinator)、 あいだにスペースをはさんで生成します。

さらに toプロパティは、 生成したセレクター文字列を引数に document.querySelectorAll() メソッドを呼び出し、 Element を選択します。 <span class="name"></span> が選択されます。

.text()メソッドで、 選択した Element の textContent に party.person.name の値を設定するよう、結びつけます。
